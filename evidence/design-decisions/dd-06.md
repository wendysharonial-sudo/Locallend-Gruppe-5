# 06: Verwendung von Jinja2-Template-Vererbung vs. Wiederholung des HTML-Codes auf jeder Seite

## Meta

Status
: **Entschieden**

Aktualisiert
: 23. Juni 2026

## Problemstellung

Unsere Webanwendung besteht aus mehreren Seiten (Landingpage, Login, Registrierung, Items, Profil etc.), die alle dieselbe Navigationsleiste, dieselben CSS-Dateien und dasselbe allgemeine Layout teilen.

Wir mussten entscheiden, ob wir diese gemeinsamen HTML-Elemente in jeder einzelnen Template-Datei wiederholen oder ein gemeinsames Basis-Template nutzen und die Jinja2-Template-Vererbung (Template Inheritance) anwenden.

## Entscheidung

Wir haben uns für die Verwendung eines Basis-Templates (`base.html`) in Kombination mit der Jinja2-Template-Vererbung entschieden.

Alle Unterseiten verwenden den folgenden Jinja2-Befehl:

{% raw %}
```jinja
{% extends "base.html" %}
```
{% endraw %}

Der eigentliche Seiteninhalt wird innerhalb des folgenden Blocks definiert:

{% raw %}
```jinja
{% block content %}
...
{% endblock %}
```
{% endraw %}

Dieser Ansatz vermeidet redundanten Code (Code-Duplizierung), sorgt für ein durchgängig konsistentes Design und ermöglicht es, seitenübergreifende Elemente wie die Navigationsleiste oder den Footer zentral an einer Stelle zu verwalten und bei Bedarf zu ändern.

*Entscheidung getroffen von:* Jean Yves Nkwane

## Betrachtete Optionen

Wir haben zwei Optionen einander gegenübergestellt:

- Vollständige Wiederholung der HTML-Struktur in jedem einzelnen Template.
- Nutzung einer gemeinsamen `base.html` gepaart mit Jinja2-Template-Vererbung.

| Kriterium | HTML wiederholen | Template-Vererbung |
| --- | --- | --- |
| **Code-Duplizierung** | ❌ Derselbe `<head>`-Bereich und dieselbe Navbar in jeder Datei | ✔️ Wird nur ein einziges Mal in der `base.html` definiert |
| **Konsistenz** | ❌ Hohes Risiko für inkonsistente Layout-Änderungen | ✔️ Eine zentrale Änderung wird sofort auf alle Seiten angewendet |
| **Wartbarkeit** | ❌ Sehr aufwendig, da 8+ Dateien manuell aktualisiert werden müssten | ✔️ Hervorragend, da nur die `base.html` angepasst werden muss |
| **Einfachheit** | ✔️ Konzeptuell simpler, da man `extends`/`block` nicht lernen muss | ❌ Erfordert ein initiales Verständnis der Jinja2-Vererbungslogik |
