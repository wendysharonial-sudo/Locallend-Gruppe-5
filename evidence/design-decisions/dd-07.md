# DD-07: Bereitstellung von Daten über eine JSON-API

## Meta

Status
: Work in progress - **Decided** - Obsolete

Updated
: 23-Jun-2026

## Problem Statement

Für LocalLend musste entschieden werden, wie Gegenstands- und Anfrageinformationen technisch bereitgestellt werden.

Die zentrale Frage war, ob Daten ausschließlich über HTML-Seiten angezeigt oder zusätzlich über eine strukturierte JSON-API bereitgestellt werden sollen.

Da das Projekt eine Flask-Webanwendung ist und mindestens ein „headless“ API-Endpunkt gefordert ist, musste eine Lösung gewählt werden, die unabhängig vom sichtbaren Frontend funktioniert.

Außerdem sollten Daten wie Gegenstände, Kategorien, Verfügbarkeit und Ausleihanfragen einfach testbar und dokumentierbar sein.

## Decision

Wir haben uns dafür entschieden, ausgewählte Daten über eine JSON-API bereitzustellen.

Dazu wurden API-Endpunkte erstellt, die Daten im JSON-Format zurückgeben. Dadurch können Gegenstände und Anfrageinformationen unabhängig von HTML-Seiten abgerufen werden.

Die JSON-API macht es einfacher, Backend-Daten zu testen, zu dokumentieren und später eventuell auch für andere Clients wiederzuverwenden.

*Decision was taken by:* Maryam

## Regarded options

Wir haben zwei Alternativen betrachtet:

+ Daten nur über HTML-Seiten anzeigen
+ Daten zusätzlich über eine JSON-API bereitstellen

| Kriterium | Nur HTML-Seiten | JSON-API |
| --- | --- | --- |
| **Projektanforderung** | ❌ Erfüllt die API-Anforderung nicht vollständig | ✔️ Erfüllt die Anforderung eines headless API-Endpunkts |
| **Testbarkeit** | ❌ Daten sind schwerer separat zu testen | ✔️ API-Endpunkte können direkt getestet werden |
| **Strukturierte Daten** | ❌ Daten sind mit HTML vermischt | ✔️ Daten werden klar strukturiert zurückgegeben |
| **Dokumentation** | ❌ Schnittstelle ist schwerer zu beschreiben | ✔️ Endpunkte und Responses können klar dokumentiert werden |
| **Wiederverwendbarkeit** | ❌ Stark an Frontend gebunden | ✔️ Kann später auch für andere Clients genutzt werden |

## Proof of regarded options

+ Datei `app.py`

## Consequences

Durch die JSON-API können Daten unabhängig vom Frontend bereitgestellt werden.

Das verbessert die Testbarkeit und macht die technische Umsetzung besser nachvollziehbar.

Die Herausforderung besteht darin, die API-Endpunkte konsistent zu dokumentieren und darauf zu achten, dass die JSON-Antworten eine klare Struktur haben.

Für LocalLend überwiegen die Vorteile, weil die API-Anbindung ein wichtiger Teil der Projektanforderungen ist und zeigt, wie Frontend und Backend technisch zusammenarbeiten.
