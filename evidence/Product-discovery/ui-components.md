# UI Components

## Ziel der Benutzeroberfläche

Die Benutzeroberfläche von LocalLend soll einfach, übersichtlich und verständlich sein.

Nutzer sollen schnell erkennen können:

- welche Gegenstände verfügbar sind
- wie sie eine Anfrage stellen können
- welchen Status eine Anfrage hat
- welche Aktionen möglich sind

---

## Wichtige UI-Komponenten

| Komponente | Zweck |
|---|---|
| Navigation | Ermöglicht den Wechsel zwischen Seiten |
| Gegenstandskarte | Zeigt wichtige Informationen zu einem Gegenstand |
| Suchfeld | Hilft beim Finden bestimmter Gegenstände |
| Kategorieanzeige | Gruppiert Gegenstände übersichtlich |
| Anfrage-Button | Ermöglicht das Erstellen einer Ausleihanfrage |
| Statusanzeige | Zeigt den aktuellen Status einer Anfrage |
| Login-Formular | Ermöglicht die Anmeldung |
| Registrierungsformular | Ermöglicht die Erstellung eines Nutzerkontos |

---

## Gegenstandskarte

Eine Gegenstandskarte zeigt kompakt die wichtigsten Informationen.

Beispiele:

| Feld | Bedeutung |
|---|---|
| Name | Name des Gegenstands |
| Kategorie | Art des Gegenstands |
| Status | Verfügbarkeit |
| Aktion | Anfrage stellen oder Details ansehen |

---

## Anfrage-Status

| Status | Bedeutung |
|---|---|
| pending | Anfrage wurde erstellt und wartet auf Entscheidung |
| accepted | Anfrage wurde angenommen |
| rejected | Anfrage wurde abgelehnt |

---

## Design-Ziel

Das Design soll besonders einfach bleiben, damit Nutzer ohne lange Erklärung verstehen, wie sie Gegenstände finden, anfragen oder verwalten können.
