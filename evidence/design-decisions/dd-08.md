# DD-08: Anfrageverwaltung mit Statuswerten

## Meta

Status
: Work in progress - **Decided** - Obsolete

Updated
: 23-Jun-2026

## Problem Statement

Für LocalLend musste entschieden werden, wie Ausleihanfragen verwaltet werden sollen.

Eine Anfrage ist nicht sofort abgeschlossen, nachdem sie erstellt wurde. Zuerst wartet sie auf eine Entscheidung des Verleihers. Danach kann sie angenommen oder abgelehnt werden.

Die zentrale Frage war, ob erledigte Anfragen gelöscht werden sollen oder ob jede Anfrage mit einem Status gespeichert bleibt.

Da der Ausleihprozess nachvollziehbar bleiben soll, wurde eine Lösung benötigt, die den aktuellen Zustand einer Anfrage klar darstellt.

## Decision

Wir haben uns dafür entschieden, Ausleihanfragen mit Statuswerten zu verwalten.

Eine neue Anfrage erhält zunächst den Status `pending`.

Wenn der Verleiher die Anfrage annimmt, wird der Status auf `accepted` geändert.

Wenn der Verleiher die Anfrage ablehnt, wird der Status auf `rejected` geändert.

Dadurch bleibt der gesamte Anfrageprozess transparent und nachvollziehbar.

*Decision was taken by:* Maryam

## Regarded options

Wir haben zwei Alternativen betrachtet:

+ Erledigte Anfragen löschen
+ Anfragen speichern und über Statuswerte verwalten

| Kriterium | Anfragen löschen | Statusverwaltung |
| --- | --- | --- |
| **Nachvollziehbarkeit** | ❌ Vergangene Entscheidungen sind nicht sichtbar | ✔️ Anfrageverlauf bleibt verständlich |
| **Transparenz** | ❌ Nutzer sehen nicht klar, was passiert ist | ✔️ Nutzer sehen pending, accepted oder rejected |
| **Debugging** | ❌ Fehler sind schwerer nachzuvollziehen | ✔️ Status kann einfach geprüft werden |
| **Workflow** | ❌ Kein klarer Lebenszyklus | ✔️ Klarer Ablauf von pending zu accepted/rejected |
| **Benutzerverständnis** | ❌ Anfrage verschwindet möglicherweise plötzlich | ✔️ Nutzer erkennen den aktuellen Stand |

## Proof of regarded options

Die Entscheidung wird durch folgende Projektergebnisse nachgewiesen:

+ Datei `app.py`
+ Datei `evidence/api-documentation.md`
+ Datei `evidence/api-explanation.md`
+ Datei `evidence/api-testing.md`
+ API-Endpunkt zum Erstellen einer Anfrage
+ API-Endpunkt zum Annehmen einer Anfrage
+ API-Endpunkt zum Ablehnen einer Anfrage
+ Statuswerte `pending`, `accepted`, `rejected`

## Consequences

Durch die Statusverwaltung wird der Ausleihprozess klarer und besser nachvollziehbar.

Nutzer können sehen, ob eine Anfrage noch offen, angenommen oder abgelehnt wurde.

Außerdem erleichtert die Statusverwaltung das Testen und Dokumentieren der Anfrage-Logik.

Die Herausforderung besteht darin, Statusänderungen konsistent umzusetzen, damit eine Anfrage nicht versehentlich in einem falschen Zustand bleibt.

Für LocalLend ist diese Entscheidung sinnvoll, weil Vertrauen und Transparenz zentrale Anforderungen des Projekts sind.
