# 01: User Authentication with Flask Sessions

## Meta

Status
: **Decided**

Updated
: 23-Jun-2026

## Problem Statement

Should we manage logged-in users by using Flask Sessions or by treating every page independently without storing user information?

Our application LocalLend requires users to register, log in and access user-specific functionality such as profile pages and item creation.

Without a mechanism to identify users across multiple requests, it would not be possible to provide personalized functionality.

## Decision

We decided to use Flask Sessions.

After a successful login, the user ID and user name are stored in the session. Protected pages can then verify whether a user is logged in.

This approach is simple, lightweight and sufficient for the scope of our project.

*Decision was taken by:* github.com/tubaclk1912

## Regarded options

We regarded two alternative options:

+ Flask Sessions
+ No session management

| Criterion | Flask Sessions | No Session Management |
| --- | --- | --- |
| **User-specific pages** | ✔️ Possible | ❌ Not possible |
| **Profile page** | ✔️ Can identify users | ❌ Same page for everyone |
| **Implementation effort** | ✔️ Low | ✔️ Low |
| **Scalability** | ✔️ Suitable for our project | ❌ Limited |
| **Logout functionality** | ✔️ Supported | ❌ Not possible |
