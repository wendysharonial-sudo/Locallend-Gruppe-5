# 05: Bootstrap über CDN oder über das Python-Paket Bootstrap-Flask laden

## Meta

Status
: **Entschieden**

Aktualisiert
: 23. Juni 2026

## Problemstellung

Unsere Webanwendung benötigt Bootstrap für responsives Styling. 
Wir mussten entscheiden, wie wir Bootstrap in unser Projekt einbinden: 
entweder durch direktes Laden über ein CDN (Content Delivery Network) im HTML, 
oder durch die Verwendung des Python-Pakets `Bootstrap-Flask`, welches Bootstrap 
in das Flask-Backend integriert.

## Entscheidung

Wir haben uns entschieden, Bootstrap direkt über das CDN zu laden.

Der CDN-Ansatz ist einfacher einzurichten, fügt dem Projekt keine weitere 
Python-Abhängigkeit hinzu und ermöglicht es uns, jede Bootstrap-Version oder 
-Komponente direkt zu verwenden. Dies bedeutet auch, dass wir das gesamte HTML 
selbst schreiben. Das gibt uns mehr Flexibilität und fördert unseren Lerneffekt 
im Umgang mit HTML und CSS.

*Entscheidung getroffen von:* Jean Yves Nkwane

## Betrachtete Optionen

Wir haben zwei Optionen betrachtet:

+ Laden von Bootstrap CSS und JS über ein CDN in der `base.html`.
+ Verwendung des Python-Pakets `Bootstrap-Flask`.

| Kriterium | CDN | Bootstrap-Flask |
| --- | --- | --- |
| **Einfachheit der Einrichtung** | ✔️ Nur Link-Tags im HTML hinzufügen | ❌ Paket muss installiert und Flask konfiguriert werden |
| **Flexibilität** | ✔️ Volle Kontrolle über das HTML | ❌ Verwendet Makros, die HTML-Details verbergen |
| **Lerneffekt** | ✔️ Wir lernen HTML und Bootstrap direkt | ❌ Makros abstrahieren das HTML |
| **Zusätzliche Abhängigkeit** | ✔️ Kein zusätzliches Python-Paket erforderlich | ❌ Fügt `Bootstrap-Flask` zur `requirements.txt` hinzu |
| **Flash-Nachrichten** | ❌ Jinja2-Schleife muss manuell geschrieben werden | ✔️ Das Makro `render_messages()` ist sehr komfortabel |
